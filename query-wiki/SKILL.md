# 知识查询 (Query)

## 触发条件

当用户要求查询、搜索、检索、询问知识库中的信息时触发。

## 工作流程

### 1. 确定目标知识库

- **检查用户是否指定了知识库名称**
  - 如果用户明确提到"查询 xxx 知识库"或"在 xxx 中搜索"，使用指定的知识库
  - 如果用户未指定，**默认使用 `default` 知识库**
- 确认知识库路径：`C:/Users/Administrator/Desktop/知识库/<知识库名称>/`
- 如果知识库不存在，提示用户创建或检查名称

### 2. 读取索引 (index.md)

- 读取 `index.md` 了解知识框架和目录结构
- 根据用户的查询意图，定位可能相关的页面类别
- 记录相关页面的链接

### 3. 搜索相关页面

- 在 `wiki/` 目录下搜索相关页面
- 读取可能相关的 wiki 页面内容
- 提取与查询相关的信息片段

### 4. 综合回答

- 整合从多个页面获取的信息
- 生成结构化的回答，包含：
  - 直接回答用户问题
  - 引用来源（使用 [[页面名]] 格式）
  - 相关的背景信息
  - 不同观点的对比（如有矛盾）
- 回答形式可以多样化：
  - Markdown 文本
  - 对比表格
  - 列表
  - 流程图描述

### 5. 归档有价值的内容（可选）

- 如果查询结果有价值、可能复用：
  - 询问用户是否将结果归档为新的 wiki 页面
  - 如果用户同意，在 `wiki/` 下创建新页面（中文文件名）
  - 例如：`Transformer与RNN对比分析.md`
  - 更新 `index.md` 索引

### 6. 记录日志 (log.md)

- 在 `log.md` 顶部追加查询记录：
  ```markdown
  ### [YYYY-MM-DD] query | <查询主题>
  
  - 知识库：<知识库名称>
  - 查询内容：<用户问题摘要>
  - 参考页面：
    - [[页面1]]
    - [[页面2]]
  - 回答摘要：<简要描述>
  ```

### 7. 确认完成

- 向用户呈现查询结果
- 提供相关页面的链接供进一步阅读
- 询问用户是否需要深入某个方面

## 注意事项

- 优先读取 index.md 了解全局，再深入具体页面
- 回答必须附带引用来源
- 如果知识库中没有相关信息，明确告知用户
- 可以建议用户摄入相关资料来补充知识库
- 查询结果可以归档为新的 wiki 页面，实现知识复利
